# 信息
 - 信息论下界：对于一个误报率为 $\epsilon$ 的过滤器，存储 $n$ 个元素所需的理论最小空间是$O\log_2(1/\epsilon)$位
 - \[B(U,V,n)=\log\binom{|U|}{n}+n\log |V|\]
 公式计算的是在理论上，存储 \(n\) 个键值对所需的最小比特数是多少。$U$ 是待存储的元素集合，所以 $|U|$ 表示不同的键的个数，V是存储的元素可取的值域，$n$ 是当前哈希表存储的元素个数，$B$ 是信息论下界
  - \(\log ^*n\) 表示将 \(n\) 求对数直到结果小于等于 \(1\) 所需的最小次数。是一个增长极慢函数。显然，\[\log ^{\log ^*n}n=1\]

# 算法成果
设计一种新的哈希表结构，在允许插入/删除操作时间为 \(O(k)\) 的前提下，将空间浪费降至 \(O(\log^k n)\) 比特/键. $k$ 是用户可调参数（通常取 \(1\leq k \leq \log^*n\)）这意味着：每增加一个常数级的时间开销，空间浪费就下降一个“对数层级”，最终逼近常数空间浪费

# 技术核心 
 - k-kick tree 在 hashing 中的应用 
 - 哈希表的存储空间设计成树形结构
 - 一共分成 $k+1$ 层，编号从 $0$ 到 $k$，同时也表示深度，第 $i$ 层对应探测深度为 $i$ 
 - 定义一个深度函数 depth(x) 表示第 x 层的桶数量变成原来的 depth(x) 分之一
