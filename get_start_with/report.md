# 研究背景和意义
哈希表是现代数据库管理系统、缓存系统和搜索引擎中最为基础的数据结构之一。其核心目标是支持动态插入和删除操作，并保证查询的时间复杂度为 O(1)，即在理想状态下，查询、插入、删除的操作所需要的时间是常数级，不随存储数据量n的增大而增加。为了实现这一点，哈希表往往需要在空间占用上做出让步，通过额外的比特位存储信息来实现极低的时间复杂度。在要求插入、删除及查询操作均维持恒定时间复杂度（即期望时间或最坏情况时间为O(1)的前提下），早期的理论工作（Yao, 1985; Pagh, 2001）表明，在严格限制更新操作为O(1)时间且仅使用简单探测序列的模型下，每个键值对的空间冗余往往难以低于Ω(log log n)比特。
2021年的论文《关于哈希表最优时间/空间权衡》打破了这一僵局。这项研究提出了一种基于 k-kick 树的新哈希方案，证明通过适当提高插入和删除操作的时间复杂度（从 O(1) 提高到O(k)其中k是一个人为指定的具有一定上限的常数），空间浪费可以以指数形式降低至 O(log^(k) n)（log^(k) n=log(log^(k-1)n)并定义log^(0)n=n；log^(1)n=log n）位。当 k = log* n（等式右侧表示对n进行迭代求对数直到结果小于等于1所需要的次数，一般地，当k=log*n时log^(k)=1）时，空间浪费可以降低到恒定的 O(1) 比特水平，实现了理论和工程上的重大突破。
本次毕业设计旨在复现这一前沿的哈希算法，并将其集成到常见的数据库管理系统中。
研究意义如下：
理论价值，通过实验对论文理论进行验证，通过调整k的值研究k如何影响时间与空间复杂度；工程价值：减少内存使用意味着成本降低和并发能力增强，在内存有限的场景中提高现有数据库的存储效率。将其集成到数据库观察空间存储效率提升效果。


# 研究现状
哈希表作为支持几乎常数时间的插入、删除和查询操作的核心数据结构，其时间复杂度和空间复杂度之间的权衡一直是现代算法研究焦点。当前主流的哈希算法实现方式主要包括链式方法和开地址法及其变体，而他们暂时都没能摆脱论文先前提到的理论局限。
链式寻址法通过链表处理冲突和碰撞的问题，它在实现上相对简单并且支持动态扩容，但每个节点都需要存储额外的指针开销，在查询时的探测深度也相当不平衡。链表节点在内存中的非连续分布还会破坏了缓存局部性，导致更高概率的缓存缺失。
开放寻址法（如线性探测、二次探测）将所有元素存储于连续数组中，通过一个固定的探测序列函数处理冲突，当当前元素哈希结果与已存入的元素冲突时，沿着探测序列寻找下一个位置，直到不发生冲突。然而，随着负载因子α趋近于1，冲突链长度急剧增加，导致查询与插入因为要沿着长长的探测序列进行操作，时间复杂度退化。因此其难以在保证O(1)时间复杂度的同时维持极高的空间利用率。
Cuckoo Hashing，由Pagh和Rodler于2001年提出（Rasmus Pagh and Flemming Friche Rodler. Cuckoo hashing. Journal of Algorithms），通过引入多候选位置及元素重定位机制，在最坏情况下实现了O(1)的查询时间。多侯选位置改变了先前使用一个关于探测次数的探测函数生成探测序列的方式，转而采用多个哈希函数为每个元素生成多个“可存入位置”，每一个元素只会出现在属于自己的“可存入位置”中。元素重定位机制则从发现冲突退让并探测转变为发现“可存入位置”都有冲突就踢出一个元素，将该元素往属于它的其他“可存入位置”移动，重复进行直到不再冲突。这两种机制协作严格保证了任何键的查找操作只需要计算预先设置的哈希函数个数的次数，实现了常数及查询。
上述所有解决方案均未能突破前面提出的空间限制下界。而2021 年发表的论文《关于哈希表时间/空间权衡的最优方案》中提出的利用 k-kick 树结构构建哈希表方案是目前最新的突破成果。该方案将哈希表映射问题转化为一个层次化的“球到槽”问题，通过引入随机优先级和有限的不超过人为确定的k次元素移动（即“k-kick”），从理论上证明，对于任何 k >= 1，都可以实现 O(k) 时间复杂度的的哈希表元素更新（插入、删除、查询）时间和 O(log^(k) n)比特的空间浪费。该研究还证明了此权衡曲线对于所有基于“增强开放寻址”框架的哈希表是紧确的（Bender, M. A., et al. "On the Optimal Time/Space Tradeoff for Hash Tables." arXiv preprint arXiv:2111.00602 (2021).），即任何此类方案若更新时间为t，则每元素必须使用Ω(log^(t) n)位的冗余，这意味着，若要在次对数级的更新时间下实现O(1)位的空间冗余必须彻底脱离“增强开放寻址”的范式。



# 主要研究内容和拟实现的目标以及研究方法
本选题旨在复现并验证2021年提出的基于k -kick树的最优时空权衡哈希算法，并探索其在数据库存储中的实际应用潜力。具体研究内容包括以下四个目标：
数据结构的实现和算法复现：依照论文中的定义，构建基于k -kick树的分层存储结构，并实现从根节点到叶节点的动态元素分配逻辑。实现高效的kick的策略，模拟论文中的随机优先级机制，并建立在插入冲突时，如何通过k次元素移动将新元素安置或触发上层重组并完成的机制，保证O(k)的时间复杂度。实现紧凑的辅助路由结构，用于快速定位元素所在的探测层级。
数据库存储集成与适配：设计形式化的的哈希表操作接口，对外只暴露insert delete get等操作，屏蔽底层逻辑设计提高兼容性。在技术和时间允许的情况下，进阶解决多线程环境下的锁竞争问题，可尝试探索如何将存储树持久化到磁盘。
理论正确性证明与复杂度分析：基于代码实现，形式化验证算法复杂度，结合理论推导与实验数据，量化分析参数k对更新时间Time(n)和空间浪费Space(n)的具体影响，验证Space(n)≈O(log^(k) n)的正确性。
性能对比：构建包含链式哈希、线性探测、Cuckoo Hashing 等主流算法作为对照组，在相同硬件环境下进行全方位性能比对。测试高负载因子环境下、混合操作（即大量混杂插入查询删除等不同操作）环境对该哈希方案的性能进行分析。

本研究将遵循“理论研读 -> 原型开发 -> 系统集成 -> 实验验证”的技术路线。在开发环境与工具上选择采用 C++17/20 标准确保底层内存操作的精细控制，使用 CMake 作为跨平台构建工具。进行单元测试，确保各模块逻辑的正确性。选用 LevelDB 作为继承目标。LevelDB 具有代码结构清晰、LSM-Tree 架构成熟等优点。首先将基于k -kick树的哈希方案封装为独立的库对外暴露规格化接口，额外提供参数配置接口。之后进行集成，通过对数据库中相关组件函数的重写实现。
